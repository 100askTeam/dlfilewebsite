# 最终上线配置说明

以下配置按当前已知环境直接适配：

- 域名：`dl.100ask.net`
- 服务器目录：`/home1/dlfile/server_release_bundle`
- Gunicorn 监听：`127.0.0.1:5000`
- Nginx 反向代理到 Gunicorn
- systemd 服务名：`dl-download-site`

## 服务器推荐目录

```text
/home1/dlfile/server_release_bundle/
├── 发布版代码
├── Hardware/
├── Video/
├── access_stats.json          # 运行后自动生成
├── admin_auth.json            # 首次登录后自动生成
└── index.html                 # 生成后自动出现
```

## 一次性初始化

```bash
cd /home1/dlfile/server_release_bundle
cp .env.example .env
bash scripts/start.sh
```

首次运行会自动完成：

- 创建虚拟环境 `.venv`
- 安装依赖
- 生成 `.secret_key`
- 递归生成目录页
- 启动 Gunicorn

## systemd 上线

```bash
cp /home1/dlfile/server_release_bundle/deploy/systemd/dl-download-site.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable dl-download-site
systemctl restart dl-download-site
systemctl status dl-download-site
```

## Nginx 上线

```bash
cp /home1/dlfile/server_release_bundle/deploy/nginx/dl-download-site.conf /etc/nginx/conf.d/dl.100ask.net.conf
nginx -t
systemctl reload nginx
```

如果你是宝塔环境，优先改用：

- [宝塔Nginx面板部署说明.md](file:///i:/dlfile/server_release_bundle/deploy/%E5%AE%9D%E5%A1%94Nginx%E9%9D%A2%E6%9D%BF%E9%83%A8%E7%BD%B2%E8%AF%B4%E6%98%8E.md)
- [dl.100ask.net.baota.conf](file:///i:/dlfile/server_release_bundle/deploy/nginx/dl.100ask.net.baota.conf)

这个版本已经按你当前宝塔站点的配置习惯适配好了：

- `root /home1/dlfile`
- 证书路径走宝塔默认证书目录
- 日志路径走 `/www/wwwlogs/`
- 不再使用 `PHP` 与 `fancyindex`

## HTTPS 证书路径

当前 Nginx 配置默认使用：

```text
/etc/letsencrypt/live/dl.100ask.net/fullchain.pem
/etc/letsencrypt/live/dl.100ask.net/privkey.pem
```

如果你的证书不在这个目录，请修改：

- `deploy/nginx/dl-download-site.conf`

## 资源上传说明

这个发布包本身不带资源文件，你需要把这些资源目录一并上传到：

- `/home1/dlfile/server_release_bundle/Hardware/`
- `/home1/dlfile/server_release_bundle/Video/`

后台上传的新文件也会直接进入这个目录结构，并自动刷新页面。
